# The Art of the Prompt

Extensy is only as powerful as your prompt. In this guide, we'll cover the anatomy of a great request and show you how to structure your ideas to get the best possible extension.

## The Anatomy of a Great Prompt

A high-performing Extensy prompt typically follows this structure:

**[Action] + [Target] + [Behavior] + [UI/Style]**

- **Action**: What should it do? (Highlight, Hide, Summarize, Alert)
- **Target**: Where should it happen? (All sites, specific URL, specific elements)
- **Behavior**: How should it handle events? (On click, on page load, on message)
- **UI/Style**: What should it look like? (Floating badge, popup with 3 buttons, green background)

## What to Include

- **Website URLs**: If your extension is for a specific site, mention it by name.
- **Visuals**: Mention colors, positions (top-left, bottom-right), and button labels.
- **Specific Behaviors**: "Only run when I click the toolbar icon" vs. "Run automatically on page load."
- **DOM Selectors**: If you're a developer, you can even mention CSS class names or ARIA labels if known.

## What NOT to Include

- **Backend Logic**: Extensy builds Chrome extensions (client-side only). It can't host databases, manage server-side accounts, or provide API keys for you.
- **OAuth Flows**: Complex OAuth redirects that require separate server-side hosting are not supported out-of-the-box in generated code.
- **System-Level Actions**: Chrome extensions are sandboxed. They can't delete files on your hard drive, control other apps, or access system settings.

<Note>
  **Important**: Extensy is your frontend and client-side logic engine. For database hosting and server-side logic, you'll need to provide your own API endpoints.
</Note>

## Iteration Strategy

If your initial generation is 80% there, don't start from scratch. Use a refinement prompt:

> "The extension I just generated highlights prices in green. Now, please add a button to the popup that lets me 'Clear Highlights' from the current page."

<Tip>
  Refinement prompts are the fastest way to build complex, multi-featured extensions.
</Tip>
